## Overview

**3B NexoSuite** is a modular ERP and e-commerce platform built for businesses that want to manage daily operations from one connected system.

It combines:

- ERP
- E-commerce
- CRM
- Inventory
- Accounting
- Procurement
- Service operations
- Customer portal
- Trial activation and licensing
- Security hardening
- Reporting and finance controls

The system is designed for companies that need a practical business platform for online sales, stock control, customers, orders, finance, service jobs, suppliers, reporting, and secure business operations.

## Scheduled Reports Update

Scheduled reports now use a simple Gmail Compose workflow.

This avoids:

- SMTP setup
- Gmail app password requirement
- XAMPP mail configuration problems
- blocked SMTP ports on local networks

### Workflow

1. Create a scheduled report.
2. Save the schedule.
3. Click **Open Gmail Compose**.
4. Gmail opens with:
   - recipient
   - subject
   - message body
   - report link
5. User sends the email from Gmail.

### Important note

Gmail Compose cannot auto-attach files from localhost. The report link is included in the email body, and the user can open/download the report and attach it manually if needed.

## Security Update

The latest build adds an overall software security layer.

### Included protections

| Protection | Description |
|---|---|
| Secure sessions | HttpOnly cookies, SameSite=Lax, session regeneration |
| CSRF protection | Token required for POST requests |
| Auto CSRF injection | Header JavaScript adds CSRF token to POST forms |
| POST rate limit | Reduces repeated request abuse |
| Login throttling | Limits repeated login attempts |
| Security audit log | Records blocked requests and login events |
| Security Center | Admin view for security events |
| Upload hardening | File type allow-list, filename cleaning and size limit |
| Request filtering | Blocks suspicious paths and traversal attempts |
| Safer redirects | Prevents unsafe external redirect behavior |
| Security headers | Adds standard browser protection headers |
| XAMPP-safe `.htaccess` | Keeps protection while avoiding Apache 500 errors |

## XAMPP Compatibility Update

The security rules were adjusted to avoid HTTP 500 errors on XAMPP.

Removed from `.htaccess`:

- `DirectoryMatch`
- `ServerSignature`
- `php_flag`

Kept:

- directory listing protection
- sensitive file blocking
- upload execution blocking
- safe security headers

If Apache shows a 500 error, check:

```text
C:\xampp\apache\logs\error.log
```

## Trial Activation System

3B NexoSuite includes a customer-specific trial and activation system.

### Trial Limits

In trial mode, the customer can create records based on the configured license limits.

Example default limits:

| Area | Trial Limit |
|---|---:|
| Products | 5 |
| Categories | 5 |
| Customers | 5 |
| Orders | 5 |

After the limit is reached, the system displays an activation message and blocks new record creation until activation is completed.

Existing records remain safe and visible.

---

## Activation Message

Recommended message shown to customer:

```text
Please activate your 3B NexoSuite license to continue creating new records.
```

---

## Customer-Specific Request Code

Each installation generates a unique request code.

The request code can be based on:

- Installation UID
- Domain or localhost path
- Database name
- Database prefix
- Shop name
- Shop email
- App encryption key fingerprint

This makes the activation request unique for each customer installation.

---

## Safe License Control

The activation system should be non-destructive.

It should not:

- Delete customer records
- Damage database tables
- Corrupt files
- Encrypt customer data after the trial limit
- Block customers from viewing existing records

It should only block new creation after the trial limit until activation is completed.

## Deployment Flow

### Basic Deployment Steps

1. Upload `installer.php` to hosting.
2. Create a MySQL database.
3. Open the installer in the browser.
4. Select the business profile.
5. Select the module bundle.
6. Enter database details.
7. Enter store and admin details.
8. Run the installation.
9. Validate admin login.
10. Validate storefront.
11. Validate product, category, customer and order creation.
12. Validate accounting dashboard and reports.
13. Validate customer portal.
14. Validate activation loader.
15. Delete `installer.php` after installation.
